# 《剧情触发器圣典》编写计划

> 本文档是给**内容编写者**看的工作手册，说明如何编写 `triggers.md`，以及如何将其部署到 GitHub Pages。

---

## 一、文件说明

`triggers.md` 是游戏 DM（AI）的专属参考文档，玩家不应直接看到此内容。  
DM 在每局游戏开始时读取本文件，并在游戏进行中根据「世界旗帜」状态判断各事件节点的触发时机。

---

## 二、旗帜命名规范

格式：`FLAG_[相关势力或人物]_[事件简述]`

| 类型 | 示例 | 说明 |
|---|---|---|
| 布尔型 | `FLAG_丘处机_目睹施救 = TRUE` | 事件发生或未发生 |
| 计数型 | `FLAG_全真_赵志敬积怨 = 2` | 累积次数，常用于渐进式触发 |

**命名原则**：简洁、唯一、见名知意。不同路线的旗帜前缀须有区分，避免歧义。

---

## 三、节点模板

每个触发节点按以下格式编写：

```
▌节点名称

触发条件：[所需旗帜 / 地点 / 时间 / 行动]
触发方式：主动（玩家行为触发）/ 被动（DM 在满足条件的场景中自然安排）
事件内容：[场景中发生了什么，1-3句话]

玩家分叉：
  ├─ [选项A描述]
  │     → 结果：[发生了什么]
  │     → 设置旗帜：FLAG_XXX = TRUE / 数值
  │     → 进入：[下一节点名称 / 路线关闭]
  │
  └─ [选项B描述]
        → 结果：[发生了什么]
        → 旗帜：[不触发 / 或其他变化]
        → 进入：[下一节点名称 / 此路线关闭]
```

**特殊情况**：部分节点无玩家选择（强制剧情），注明"此处无分叉，自动推进至下一节点"即可。

---

## 四、路线文件结构

每条入派路线独立成块，按以下框架组织：

```
══════════════════════════════════════
【路线名称】（例：全真教入派 → 叛出 → 古墓派收留）
目标门派：XX
前置条件：无 / 需要旗帜 FLAG_XXX = TRUE
适用地图：牛家村 / 终南山 / ……
══════════════════════════════════════

▌节点1：事件名
……

▌节点2：事件名
……

▌路线终点
结果：[玩家加入了哪个门派，或有何变化]
解锁内容：[新地图 / 新剧情线 / 新武学体系]
```

## 五、完整示例：全真教 → 叛出 → 古墓派

```
══════════════════════════════════════
【全真教入派 → 叛出全真 → 古墓派收留 联动路线】
目标门派：古墓派（经由全真教过渡）
前置条件：无（开局即可触发）
适用地图：牛家村 → 终南山全真教 → 古墓入口
══════════════════════════════════════

▌节点1：官差欺辱事件

触发条件：玩家在牛家村，累计行动数 ≥ 3
触发方式：被动（DM 保证此事件必然出现，自然融入场景）
事件内容：两名官差在客栈门口欺辱一名村妇，周围村民敢怒不敢言

玩家分叉：
  ├─ [上前出手阻拦]
  │     → 结果：被官差痛打，重伤倒地
  │     → 设置旗帜：FLAG_丘处机_目睹施救 = TRUE
  │     → 进入：节点2
  │
  └─ [不插手 / 绕行 / 其他]
        → 道德值 -5
        → 旗帜：不触发
        → 此路线永久关闭


▌节点2：丘处机现身

触发条件：FLAG_丘处机_目睹施救 = TRUE
触发方式：被动（玩家倒地后，DM 在下一段场景中自然加入）
事件内容：一名灰袍道士出手打退官差，将玩家扶起简单包扎，留下一句"小兄弟有勇气"后隐入后山树林

玩家分叉：
  ├─ [追入后山树林寻访道士]
  │     → 进入：节点3
  │
  └─ [不追，在村里继续养伤]
        → 旗帜保留，节点3待玩家日后主动前往树林时再触发


▌节点3：树林相遇，丘处机邀约

触发条件：玩家主动前往后山树林 + FLAG_丘处机_目睹施救 = TRUE
触发方式：主动（玩家行为触发）
事件内容：在树林深处找到正在打坐的丘处机，他睁眼打量片刻，提出可带玩家上山入全真教

玩家分叉：
  ├─ [恭敬拜师，请求加入全真教]
  │     → 设置旗帜：FLAG_全真教_已加入 = TRUE
  │     → 进入：节点4
  │
  └─ [道谢后婉拒]
        → 设置旗帜：FLAG_丘处机_好感 = TRUE
        → 全真/古墓路线挂起，不再主动推进


▌节点4：全真教内——赵志敬考核

触发条件：FLAG_全真教_已加入 = TRUE，加入后第 1～7 天内
触发方式：被动（DM 安排赵志敬主动来审问）
事件内容：赵志敬在练功场拦住玩家，逐一审问出身、功底与来意，言辞中带有明显的轻视

玩家分叉（可多轮问答，每次冒犯性回答累加积怨值）：
  ├─ [谦逊认可的回答]
  │     → FLAG_全真_赵志敬积怨 不变
  │
  └─ [顶撞 / 含糊 / 玩笑性回答]
        → FLAG_全真_赵志敬积怨 += 1


▌节点5：比武失败——叛出全真

触发条件：FLAG_全真_赵志敬积怨 ≥ 2 + 加入全真满 60 天 + 玩家参与门内比武
触发方式：被动（DM 在满足条件时安排比武场景）
事件内容：全真门人比武，玩家输给一名师兄。赵志敬当众冷嘲热讽，言辞极为难堪

玩家分叉：
  ├─ [忍气吞声，继续留下]
  │     → FLAG_全真_赵志敬积怨 归零
  │     → 节点5可在积怨再次累积后重新触发
  │
  └─ [拂袖而去，叛出全真]
        → 设置旗帜：FLAG_叛出全真 = TRUE
        → 进入：节点6


▌节点6：下山路遇古墓（强制节点）

触发条件：FLAG_叛出全真 = TRUE + 玩家选择"下山离开"
触发方式：被动（DM 在下山途中必然安排此遭遇）
事件内容：山道上遭遇霍都，正试图强行闯入古墓石门。古墓蜂群突然涌出，同时袭击霍都与玩家，玩家被蜂蛰后毒发倒地

此处无分叉，自动推进至节点7。


▌节点7：小龙女收留

触发条件：由节点6自动进入
事件内容：玩家在朦胧中被人拖入古墓石门，醒来后发现自己躺在石室内，一名白衣女子正在为自己解毒

玩家分叉：
  ├─ [开口说话，试图了解处境]
  │     → 进入与小龙女的对话交互，古墓派路线正式开启
  │
  └─ [闭眼假装未醒，暗中观察]
        → 多一轮环境描述，之后自然进入对话


▌路线终点
结果：玩家进入古墓，开启古墓派剧情线
解锁内容：古墓内部地图、小龙女教功互动、古墓武学体系（玉女心经等）
```

## 七、GitHub Raw URL 部署步骤

### 第一步：创建仓库

1. 登录 GitHub，新建一个 **Public** 仓库（名称随意，例如 `wuxia-triggers`）。
2. 在仓库根目录创建 `triggers.md`，内容为写好的《剧情触发器圣典》。
3. 如果未来同时启用 GitHub Pages，可额外在发布源根目录创建空文件 `.nojekyll`；仅使用 Raw URL 时不需要 `.nojekyll`。

### 第二步：上传并确认分支

1. 将 `triggers.md` 提交到仓库默认分支，通常是 `main`。
2. 打开 GitHub 仓库中的 `triggers.md`。
3. 点击文件页面上的 **Raw** 按钮，确认浏览器显示的是纯文本内容。

### 第三步：获取 Raw URL

Raw URL 格式为：

```
https://raw.githubusercontent.com/[你的GitHub用户名]/[仓库名]/main/triggers.md
```

如果仓库默认分支不是 `main`，将 URL 中的 `main` 替换为实际分支名。

将此地址填入游戏引擎提示词的初始化指令中，替换 `[YOUR_USERNAME]` 和 `[YOUR_REPO]` 即可。

### 第四步：正式前测试

在 `triggers.md` 顶部保留版本标记，例如：

```
TRIGGER_CANON_VERSION: jy-2026-06-04-v1
```

正式游玩前，先在 DeepSeek 单独测试：

```
请启用联网搜索并访问这个 URL。只回答是否看到了 TRIGGER_CANON_VERSION，不要复述文件内容。
```

DeepSeek 能答出已看到版本标记，才说明链路可用。

### 第五步：日后更新

直接在 GitHub 上编辑 `triggers.md` 并提交。Raw URL 通常会很快更新，但可能存在短时间缓存；正式游玩前用版本标记再测一次即可。若 URL 中用户名、仓库名、分支名或文件路径不变，游戏引擎提示词无需再改。
