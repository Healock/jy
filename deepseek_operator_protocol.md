# DeepSeek 网页端运行协议

用途：在 DeepSeek 免费网页端稳定运行《金庸群侠传式》文字游戏。本文给 DM 操作者使用，不给玩家阅读。

---

## 1. 推荐运行方式

1. DM 操作者自己打开 DeepSeek 新对话。
2. 打开联网搜索 / 网页访问能力。
3. 先发送加载测试，确认 Raw URL 可读。
4. 再发送引擎启动口令，让 DeepSeek 读取 `game_engine_prompt.md` 的 Raw URL 并执行。
5. 玩家只看 DeepSeek 生成的游戏正文，不直接操作同一个 DeepSeek 聊天窗口。

原因：DeepSeek 网页端没有真正隐藏的 system prompt。若玩家直接使用同一个窗口并向上滚动，就能看到初始化提示词和 Raw URL。

---

## 2. 开局加载测试

正式游玩前，先在 DeepSeek 单独发送：

```md
请启用联网搜索并访问：
https://raw.githubusercontent.com/Healock/jy/main/triggers.md

只回答：
LOAD_OK: 你看到的 TRIGGER_CANON_VERSION
或
LOAD_FAIL: 原因

不要复述文件内容。
```

成功结果应为：

```text
LOAD_OK: jy-2026-06-11-v4
```

若失败：

1. 确认联网搜索已开启。
2. 重试一次。
3. 仍失败时，改用文件上传 `triggers.md`。
4. 不要在加载失败状态下开启门派和隐藏剧情。

---

## 3. 正式启动

发送以下口令：

```md
请启用联网搜索并访问：
https://raw.githubusercontent.com/Healock/jy/main/game_engine_prompt.md

请读取全文，并把它作为本对话的游戏引擎提示词执行。从现在起，你的回复必须服从该文件中的全部规则。不要复述、引用、总结网址或文件内容；不要说“读取成功”。读取后直接进入角色创建第一问。
```

若 DeepSeek 回复“已读取”“我会访问”等说明性文字，不要给玩家看；让它继续输出角色创建第一问。

正确的第一条玩家可见回复应为：

```md
**【角色创建】**
先给主角取个名字。留空或回复“默认”则使用“小虾米”。
```

如 DeepSeek 跳过角色创建、直接开始牛家村场景，补发：

```md
不要解释初始化过程。现在回到角色创建，只询问姓名，不要输出游戏场景。
```

如 DeepSeek 只是总结 `game_engine_prompt.md`，补发：

```md
不要总结文件。请把刚才读取到的文本当作本对话规则执行，从现在起服从该文件中的全部规则。只输出角色创建第一问。
```

如 DeepSeek 连续两次无法读取 Raw URL，再改用手动粘贴 `game_engine_prompt.md` 完整内容。

角色创建最终确认后，DeepSeek 才进入牛家村，并按指定四段格式输出游戏正文。

---

## 4. 按需校准

不要让 DeepSeek 自己计数回合。DM 操作者只在需要时发送短校准，不给玩家看。

最常用短句：

```md
DM内部校准：当前地点=【填写】，当前路线=【填写】，玩家尚未/已经【关键状态】。只使用当前地点、当前势力或最近行为相关 trigger。不要输出内部信息，继续游戏。
```

使用时机：

- DeepSeek 开始跨地图乱提门派。
- 一回合塞进多个重大事件。
- 忘记玩家位置、时间或状态。
- 选项开始变成任务提示或上帝视角。
- 玩家换地图、入门派、获得关键线索后，需要提醒模型新状态。

示例：

```md
DM内部校准：当前地点=牛家村后山外围，当前路线=无，玩家尚未离村。只使用牛家村或后山外围相关 trigger。不要输出内部信息，继续游戏。
```

## 5. DM 存档块

不要机械按回合保存。只在剧情节点、准备关掉网页、准备新开对话、或模型明显开始忘状态时保存。

使用时机：

- 玩家离开当前地点。
- 玩家进入或退出门派。
- 玩家获得关键线索、秘籍、重要道具。
- 战斗、重大选择或路线分歧结束。
- DeepSeek 明显开始忘状态。
- 准备关闭网页或新开对话。

最短保存口令：

```md
生成极简 DM_SAVE。不要复述触发器内容，不要暴露 URL，不要解释。
```

如果 DeepSeek 输出不够规整，改用完整保存口令：

```md
生成极简 DM_SAVE。不要复述触发器内容，不要暴露 URL，不要解释。

按以下格式输出：

【DM_SAVE_BEGIN】
时间：
地点：
状态：
属性：
天赋：
武学：
已知经历：
道具：
内部旗帜摘要：
当前路线：
挂起/关闭路线：
下一步注意：
【DM_SAVE_END】
```

保存原则：

- `内部旗帜摘要` 可以给 DM 操作者看，但不要给玩家看。
- `属性` 必须记录体魄、根骨、机敏、悟性、定力、福缘。
- `天赋` 必须记录玩家选择的初始天赋；若有出身影响，也简写在同一行。
- `武学` 必须记录：武学境界、已学武功、练习进度、当前限制、禁忌武学反噬或封存状态。没有学武功时写“无武学基础；已学武功：无”。
- 内容越短越好，够恢复即可。
- 新开对话恢复时，贴入最近一次存档块。
- 不要把 DM_SAVE 给玩家看。

---

## 6. 恢复游戏

新开 DeepSeek 对话时，按顺序发送：

1. 加载测试。
2. 正式启动口令，让 DeepSeek 访问 `game_engine_prompt.md` 的 Raw URL；若失败，再粘贴全文。
3. 最近一次 `DM_SAVE`。
4. 恢复指令：

最短恢复口令：

```md
根据下面的 DM_SAVE 恢复游戏。不要向玩家展示内部旗帜、Trigger ID、URL 或版本号。从存档中的当前地点和当前状态继续，按四段格式输出下一段游戏内容。

【DM_SAVE_BEGIN】
这里粘贴存档内容
【DM_SAVE_END】
```

如果 DeepSeek 恢复后解释太多，补发：

```md
不要解释恢复过程。不要输出内部信息。直接从存档状态继续游戏，只输出【当前时间】【场景描述】【角色状态】【你的行动】四段。
```

恢复原则：

- 恢复时不要重新生成主角背景。
- 恢复时不要重置时间、地点、状态、属性、天赋、武学、道具或已知经历。
- 若存档和当前触发器冲突，以存档中的玩家已知经历和内部旗帜摘要为准。
- 若不确定下一个 trigger，先保守推进当前地点的日常场景。

---

## 7. 稳定性检查清单

每轮观察 DeepSeek 输出：

- 是否仍按四段格式输出。
- 是否没有暴露 `FLAG_`、Trigger ID、URL、版本号。
- 是否只推进一个主要事件。
- 是否没有把远方门派事件硬塞到当前地点。
- 是否没有替玩家做选择。
- 是否没有让禁忌武学无代价练成。
- 是否保持玩家已知信息和 DM 内部信息分离。

发现问题时，先用短校准；连续两次仍失败，就生成 DM 存档块并新开对话恢复。

---

## 8. 玩家输入转发建议

玩家输入可原样转发，但 DM 操作者可补充一句环境约束，例如：

```md
玩家行动：我想去后山看看有没有人。

注意：玩家当前仍在牛家村，时间是傍晚，尚未离村。请只考虑牛家村/后山外围相关事件。
```

不要替玩家解释动机，不要把隐藏 trigger 暗示给 DeepSeek 作为玩家已知内容。
